# Build the Case File

**The job:** Open a new CID case note from whatever thread starts it, and keep it growing as the case develops — so Tom does the police work and the file writes itself.

## Boot chain (read these, in order, and you have the skill)
1. This note, end to end.
2. [[Investigative Method]] — the four threads and the working rules. The starting thread decides the structure of the first moves.
3. [[Active Priorities]] — where open cases are tracked.
4. The [[CID]] index — to register the new case.

## The procedure
1. Ask Tom for the starting thread: a plate, a name, an appearance/description, or an area+pattern. One line is enough to open the file.
2. Create the case note in `02 - SAHP/CID/` named `Case - [short handle].md` with sections: **Thread** (what started it) · **Facts on record** · **Persons & vehicles** · **Evidence log** · **Working notes (theories, clearly marked)** · **Next moves**.
3. Fill in what's known. Facts under Facts; hunches under Working notes, never mixed.
4. Build "Next moves" from the Investigative Method: given this thread, what gets pulled next — and list the threads *not yet tried*.
5. Register the case in the [[CID]] index and add a line to [[Active Priorities]].
6. On every update Tom gives, log it, update Next moves, and flag what he might have missed.

## Quality bar
- Facts and theories never mix. A future reader can tell at a glance what's on record and what's a hunch.
- Dead ends are logged, not deleted — today's dead end is tomorrow's cross-reference.
- "Next moves" is never empty on an open case. If it is, the job is to say so and work the four threads for a new angle.
- Case closed → confirm with Tom, then archive per the vault rules.
