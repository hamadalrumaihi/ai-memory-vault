# Write the Incident Report

**The job:** Draft the official incident report after a callout, stop, or arrest, in department format, ready for Tom to review and submit.

## Boot chain (read these, in order, and you have the skill)
1. This note, end to end.
2. The Writing Rules in [[VAULT-INDEX]] — department format, facts only, 24-hour times.
3. The related case note in `CID/`, if the incident belongs to an open case — so the report lines up with what's already on record.

## The procedure
1. Get the raw account from Tom: what happened, when, where, who. Pull anything already logged (MDT entries, bodycam timestamps) he provides.
2. Build the header block: date, time, location, unit, case/incident number, officers involved.
3. Write the narrative: chronological, first person, observed facts and direct statements only. No theories, no speculation, no filler.
4. List evidence collected and its disposition, persons involved with full name and role on first mention, and charges if any.
5. Read it back against Tom's account. Flag anything missing or inconsistent — a gap in the timeline, an unnamed witness, evidence mentioned but not logged.
6. Save the draft to the relevant case folder (or `CID/` if standalone) as `Report YYYY-MM-DD [incident].md` and tell Tom it's ready. Never file or submit it — Tom submits.

## Quality bar
- Department format, complete header block, chronological narrative.
- Zero speculation in the narrative. If Tom's account includes a theory, it stays out of the report.
- Every person, vehicle, and item of evidence mentioned is fully identified.
- Ends with the missed-angles check: anything in the account that the report doesn't cover.
